**Create js/phaseLock.js** — a new module that computes phase lock score using Pattern 2 from RESEARCH.md (FFT bin extraction, no IFFT needed).

Module shape:
```javascript
// js/phaseLock.js
import { AppState } from './state.js';

// Shared FFT instance — set by initPhaseLock()
let _fft = null;

// Constants
const SAMPLE_RATE_HZ = 4;
const FFT_SIZE = 512;
const MIN_WINDOW_MS = 25000;  // 25 seconds minimum data before producing a score
const RSA_HALF_BAND = 0.02;   // Hz, band around pacing freq for amplitude gate

export function initPhaseLock(fftInstance) {
  _fft = fftInstance;
}
```

**computePhaseLockScore(windowSeconds, pacingFreqHz):**

1. Collect RR intervals from AppState.rrBuffer/rrHead/rrCount going backward until `windowSeconds * 1000` ms accumulated (same pattern as `computeSpectralRSA` in dsp.js). Use `windowSeconds = 30` default.
2. If accumulated time < MIN_WINDOW_MS, set `AppState.phaseLockCalibrating = true`, `AppState.phaseLockScore = 0`, return null.
3. Set `AppState.phaseLockCalibrating = false`.
4. Build evenly-spaced tachogram via cubic spline interpolation at SAMPLE_RATE_HZ (reuse the same algorithm as `buildEvenlySpacedTachogram` but inline the steps since we need the FFT output, not just the tachogram). Steps:
   - Convert RR intervals to cumulative time and instantaneous HR
   - Cubic spline interpolate to evenly-spaced samples at 4 Hz
   - Subtract mean (detrend)
   - Apply Hann window
5. FFT: `_fft.realTransform(fftOut, tachogram)`
6. Extract complex coefficient at the pacer frequency bin:
   ```javascript
   const bin = Math.round(pacingFreqHz * FFT_SIZE / SAMPLE_RATE_HZ);
   const re = fftOut[2 * bin];
   const im = fftOut[2 * bin + 1];
   const hrPhase = Math.atan2(im, re);
   ```
7. Compute pacer phase analytically. The pacer runs at `pacingFreqHz` starting from `AppState.pacerEpoch`. Compute elapsed time from epoch to the center of the data window:
   ```javascript
   // The data window covers the last `accMs` milliseconds
   // Center of window in real time:
   const windowCenterTime = (Date.now() - accMs / 2) / 1000;
   const elapsedFromEpoch = windowCenterTime - AppState.pacerEpoch / 1000;
   const pacerPhase = (2 * Math.PI * pacingFreqHz * elapsedFromEpoch) % (2 * Math.PI);
   ```
   NOTE: If `AppState.pacerEpoch` is not available or is 0, use a simpler approach — just measure phase error relative to a fixed reference (this makes the absolute phase meaningless but the relative stability still works). In that case, track the previous hrPhase and compute consistency of phase over time as a surrogate.

   ACTUALLY, the simpler and more robust approach per RESEARCH.md Pattern 2: Since we're comparing HR phase to pacer phase within the same window, and the FFT phase is relative to the window start, compute the expected pacer phase at the window center relative to window start:
   ```javascript
   const windowCenterSec = accMs / 2000;
   const pacerPhase = (2 * Math.PI * pacingFreqHz * windowCenterSec) % (2 * Math.PI);
   ```
   This avoids needing AudioContext epoch. The FFT phase is relative to the data window, and the pacer frequency defines the expected phase progression within that window.

8. Phase error: circular distance
   ```javascript
   let err = hrPhase - pacerPhase;
   if (err > Math.PI)  err -= 2 * Math.PI;
   if (err < -Math.PI) err += 2 * Math.PI;
   ```

9. Score: `(1 - |err| / PI) * 100`, clamped to [0, 100], rounded.

10. Amplitude gate: Build PSD from the FFT output (same as computePSD in dsp.js — just compute power at the pacing freq band). If power at `pacingFreqHz ± RSA_HALF_BAND` is below a minimum threshold, return 0. **Start with threshold = 0 (disabled)** per RESEARCH.md recommendation — tune empirically after first real session. Add a `const MIN_POWER_THRESHOLD = 0;` constant that can be adjusted.

11. Write score to `AppState.phaseLockScore`.
12. Export both `initPhaseLock` and `computePhaseLockScore`.

**IMPORTANT implementation notes:**
- `buildEvenlySpacedTachogram()` in dsp.js returns a zero-padded Float32Array of FFT_SIZE. The phaseLock module needs the same — either import and reuse it, or replicate the pattern. Since phaseLock needs to run its own FFT on the tachogram (independent of the one dsp.js tick already ran), the cleanest approach is: import `buildEvenlySpacedTachogram` from dsp.js and reuse it, then run FFT on the result. The dsp.js already exports it.
- The Hann window function in dsp.js (`applyHannWindow`) is NOT exported. Either export it from dsp.js, or inline a simple Hann window in phaseLock.js (it's 4 lines of code — inlining is fine).
- `computePSD` is NOT exported from dsp.js. For the amplitude gate, compute the power inline: `const power = (re*re + im*im)` at the pacing freq bin. This is simpler than building a full PSD — we only need power at one bin.
