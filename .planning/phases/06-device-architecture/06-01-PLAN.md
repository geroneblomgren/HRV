<task type="auto">
  <name>Task 3: Create DeviceManager orchestrator</name>
  <files>js/devices/DeviceManager.js</files>
  <action>
Create `js/devices/DeviceManager.js` — the orchestration layer that holds adapter slots, manages HR source priority, and derives backward-compatible AppState fields.

**Imports:** `AppState, subscribe` from `../state.js`. HRMAdapter functions from `./HRMAdapter.js`.

**Adapter slots:**
```javascript
const _adapters = {
  chestStrap: { connect: hrmConnect, disconnect: hrmDisconnect, getCapabilities: hrmCaps, getDeviceType: hrmType },
  muse: null  // Phase 7 wires this
};
```

**Exported functions:**

1. `init()` — async. Subscribe to `chestStrapConnected` and `museConnected` to:
   - Derive `AppState.connected` = OR of both device connected states
   - Derive `AppState.connectionStatus` from the primary device status (chest strap preferred, fall back to muse)
   - Call `_updateHRSource()` on any change
   - Call `_updateCapabilities()` on any change

2. `connectChestStrap()` — async. Calls `_adapters.chestStrap.connect()`. Updates `AppState.chestStrapCapabilities` from `getCapabilities()`.

3. `connectMuse()` — async. Phase 6 stub: calls `navigator.bluetooth.requestDevice({ filters: [{ namePrefix: 'Muse' }], optionalServices: [0xfe8d] })` to open the picker (so Phase 7 does not need to re-pair), but after picker selection just stores the device name via `setSetting('museName', device.name)` and sets `AppState.museName = device.name`. Does NOT attempt GATT connect (no adapter yet). Mark the Muse button with "(Beta)" label via AppState.

4. `disconnectAll()` — calls disconnect on each connected adapter.

**Internal functions:**

`_updateHRSource()` — HR source priority logic:
- If `hrSourceLocked` is true (session active), do NOT change `hrSourceLabel`
- If chest strap connected and has RR capability: `hrSourceLabel = 'Chest Strap'`
- Else if muse connected and has PPG capability: `hrSourceLabel = 'Muse PPG'`
- Else: `hrSourceLabel = null`

`_updateCapabilities()` — Merge capabilities from all connected devices. Write combined capabilities to a new `AppState.activeCapabilities` field (logical OR of each flag across connected devices). This tells UI what data is available from ANY source.

**Backward compatibility:**
- `AppState.connected` derived as `chestStrapConnected || museConnected`
- `AppState.connectionStatus` derived from chest strap status when connected; otherwise from muse status; otherwise 'disconnected'
- `AppState.savedDeviceName` mirrored from `AppState.chestStrapName` for any code still reading it
