Create js/dsp.js as an ES module with these components:

**Constants:**
- SAMPLE_RATE_HZ = 4 (standard HRV resampling rate)
- MIN_WINDOW_SECONDS = 120 (calibration duration)
- FFT_SIZE = 512 (next power of 2 >= 4Hz * 120s)
- COHERENCE_WINDOW_SECONDS = 64 (HeartMath standard)
- LF_LOW_HZ = 0.04, LF_HIGH_HZ = 0.15, TOTAL_HIGH_HZ = 0.26
- PEAK_WINDOW_HZ = 0.030 (HeartMath: integrate +/-0.015 Hz around peak)

**cubicSplineInterpolate(xs, ys, x):** ~25-line natural cubic spline interpolation. Given sorted arrays xs and ys, returns the interpolated y value at query point x. Uses tridiagonal system solve for spline coefficients.

**buildEvenlySpacedTachogram(rrBuffer, rrHead, rrCount):**
1. Read `Math.min(rrCount, 512)` values from circular buffer in chronological order (oldest first)
2. Build cumulative time axis from RR intervals (each RR is the time since previous beat)
3. Resample to SAMPLE_RATE_HZ using cubicSplineInterpolate — convert RR intervals (ms) to instantaneous HR (BPM = 60000/rr) before resampling
4. Return Float32Array(FFT_SIZE) zero-padded
5. Return null if count < 10

**applyHannWindow(signal):** In-place Hann window: `signal[i] *= 0.5 * (1 - cos(2*PI*i/N))`

**computePSD(tachogram):**
1. Initialize FFT instance from fft.js: `new FFT(FFT_SIZE)`
2. Transform tachogram using `fft.realTransform(output, tachogram)`
3. Convert complex output to power spectral density: `psd[i] = re*re + im*im`
4. Return Float32Array of PSD values indexed by frequency bin

**Helper: binToHz(bin):** `bin * SAMPLE_RATE_HZ / FFT_SIZE`
**Helper: hzToBin(hz):** `Math.round(hz * FFT_SIZE / SAMPLE_RATE_HZ)`

**integrateBand(psd, lowHz, highHz):** Sum PSD values from hzToBin(lowHz) to hzToBin(highHz).

**findPeakBin(psd, lowHz, highHz):** Return bin index with maximum PSD value in the given frequency range.

**computeCoherenceScore(psd):** HeartMath-derived formula:
1. Find dominant peak in 0.04-0.26 Hz range
2. Integrate +/-0.015 Hz window around peak (peakPower)
3. Compute power below and above the peak window
4. Coherence Ratio: CR = (peakPower / powerBelow) * (peakPower / powerAbove) — use Math.max(denominator, 0.001) to avoid division by zero
5. Coherence Score: CS = Math.log(CR + 1)
6. Map to 0-100: Math.min(100, Math.round((CS / 3.0) * 100))

**getHRArray(windowSeconds):** Read the last `windowSeconds` worth of RR intervals from the circular buffer, convert each to BPM (60000/rr), return as a regular Array. This is what WaveformRenderer will call each frame. Default windowSeconds = 60.

**computeRSAAmplitude(hrSamples):** `Math.round((Math.max(...hrSamples) - Math.min(...hrSamples)) * 10) / 10`. Returns 0 for empty array.

**initDSP():** Create the FFT instance. Called once from main.js after fft.js CDN script has loaded. Store in module-level `_fft` variable.

**tick(sessionElapsedSeconds):** The main entry point called once per second by session controller:
1. If sessionElapsedSeconds < MIN_WINDOW_SECONDS: set AppState.calibrating = true, return early
2. Set AppState.calibrating = false
3. Call buildEvenlySpacedTachogram — if null, return
4. Apply Hann window
5. Compute PSD
6. Write AppState.spectralBuffer = psd
7. Write AppState.lfPower = integrateBand(psd, LF_LOW_HZ, LF_HIGH_HZ)
8. Write AppState.coherenceScore = computeCoherenceScore(psd)

**fft.js import:** Import from CDN using a dynamic import or assume fft.js is loaded as a global via a script tag in index.html. Per Phase 1 patterns, use `<script src="https://cdn.jsdelivr.net/npm/fft.js@4.0.4/lib/fft.js"></script>` in index.html (non-module script that exposes global `FFT`). Reference `FFT` as a global in dsp.js.

Also create js/dsp.test.html — a minimal HTML test harness that loads dsp.js and runs assertions using console.assert. Include synthetic test data (known sine wave RR intervals, known PSD arrays) to verify the pipeline. This is NOT a production file; it is for verification only.

- Do NOT apply FFT before 120s elapsed
