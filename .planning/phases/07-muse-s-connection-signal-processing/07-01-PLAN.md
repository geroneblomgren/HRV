**js/devices/MuseAdapter.js** — Create new file implementing the DeviceAdapter interface. Structure:

1. **Constants** — All UUIDs from research (MUSE_SERVICE=0xfe8d, MUSE_CONTROL_UUID, EEG_UUIDS[4], PPG_UUIDS[3]). Use exact UUIDs from 07-RESEARCH.md Code Examples section.

2. **encodeCommand(cmd)** — Exact muse-js encoding: `const encoded = new TextEncoder().encode('X' + cmd + '\n'); encoded[0] = encoded.length - 1; return encoded;`

3. **decodeUnsigned12BitData(bytes)** — Parse packed 12-bit EEG samples. For each 3 bytes: `a = (bytes[i] << 4) | (bytes[i+1] >> 4)` and `b = ((bytes[i+1] & 0xF) << 8) | bytes[i+2]`. Returns array of 12 raw values.

4. **parseEEGNotification(event, channelIndex)** — Skip 2-byte event index, decode 12-bit packed data, convert to microvolts via `0.48828125 * (n - 0x800)`. Write samples into `AppState.eegBuffers[channelIndex]` circular buffer at `AppState.eegHead`. Only increment eegHead once per notification batch (all 4 channels share the same head pointer — but since notifications arrive per-channel, increment by 12 samples per channel notification). Actually, each channel gets its own circular buffer but they share timing. Write 12 samples starting at `AppState.eegHead % 512` for that channel. Increment eegHead by 12 ONLY from channel 0 (TP9) to keep a single time reference.

5. **parsePPGNotification(event, channelIndex)** — Skip 2-byte event index, parse 6 x 24-bit unsigned samples (3 bytes each, big-endian: `(bytes[i] << 16) | (bytes[i+1] << 8) | bytes[i+2]`). Write to `AppState.ppgDebugBuffers[channelIndex]` circular buffer. Increment ppgDebugHead by 6 ONLY from channel 0.

   **CRITICAL: PPG samples are 24-bit (3 bytes each), NOT 16-bit.** This was a correction from prior research.

6. **connect()** — Full GATT connection sequence:
   - If `_device` exists (reconnect), attempt `_device.gatt.connect()` with Promise.race 12s timeout (same pattern as HRMAdapter)
   - If no device, check `navigator.bluetooth.getDevices()` for saved museName, fall back to `requestDevice({ filters: [{services: [0xfe8d]}] })`
   - Set `AppState.museStatus = 'connecting'`
   - Get primary service (0xfe8d)
   - Fetch control characteristic + all EEG + all PPG characteristics via `Promise.all()`
   - Subscribe ALL characteristics (`startNotifications()` + `addEventListener`) BEFORE sending commands
   - Attach `_handleEEG(event, channelIndex)` and `_handlePPG(event, channelIndex)` listeners
   - Send command sequence: `h` → `p50` → `s` → `d` (each via `controlChar.writeValue(encodeCommand(...))`)
   - Set `AppState.museConnected = true`, `AppState.museStatus = 'streaming'`, `AppState.museName = device.name`, `AppState.museCapabilities = { hr: true, rr: true, eeg: true, ppg: true }`
   - Register `gattserverdisconnected` listener (with removeEventListener first to prevent accumulation)
   - Save device name: `setSetting('museName', device.name)`

7. **disconnect()** — Clean up all characteristic event listeners, call `device.gatt.disconnect()`, reset AppState Muse fields to disconnected defaults. Reset eegHead, ppgDebugHead, eegBuffers, ppgDebugBuffers.

8. **getCapabilities()** — Return `{ hr: true, rr: true, eeg: true, ppg: true }`

9. **getDeviceType()** — Return `'muse-s'`

10. **onDisconnected()** — Set `AppState.museConnected = false`, `AppState.museStatus = 'disconnected'`. Clean up characteristic listeners. No auto-reconnect for Muse in Phase 7 (simpler; can add later).

11. **PPG callback hook** — Expose a way for Plan 02's PPG pipeline to register a callback that receives filtered PPG samples per-channel. Use a simple module-level variable: `let _ppgCallback = null; export function setPPGCallback(fn) { _ppgCallback = fn; }`. In parsePPGNotification, after writing to debug buffers, call `_ppgCallback(channelIndex, samples)` if set.

12. **EEG callback hook** — Same pattern: `let _eegCallback = null; export function setEEGCallback(fn) { _eegCallback = fn; }`. Called after writing to eegBuffers.

Import `AppState` from `../state.js` and `getSetting, setSetting` from `../storage.js`.
