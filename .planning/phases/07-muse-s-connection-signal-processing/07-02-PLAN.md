Create `js/museSignalProcessing.js` with the complete PPG processing pipeline:

**1. Butterworth Bandpass Filter (0.5-3 Hz at 64 Hz Fs)**

Implement a 4th-order Butterworth bandpass using two cascaded biquad sections (Transposed Direct Form II):

```javascript
function makeBiquad(b0, b1, b2, a1, a2) {
  let z1 = 0, z2 = 0;
  return function process(x) {
    const y = b0 * x + z1;
    z1 = b1 * x - a1 * y + z2;
    z2 = b2 * x - a2 * y;
    return y;
  };
}
```

Pre-compute the filter coefficients for 0.5-3 Hz bandpass at 64 Hz sample rate. Use scipy.signal.butter(2, [0.5/32, 3/32], btype='band', output='sos') or equivalent. The executor MUST compute these coefficients (e.g., using a quick Python snippet or the EarLevel biquad calculator) and hardcode them. Do NOT use placeholder values. The filter has two second-order sections, each needing b0, b1, b2, a1, a2.

Create `makeBandpass()` that returns a function chaining two biquads.

**2. Peak Detection (derivative + adaptive threshold)**

Implement sample-by-sample peak detection:
- Track previous filtered sample and previous derivative
- Detect zero-crossing of derivative (positive to negative) where filtered sample exceeds adaptive threshold
- Refractory period: minimum 300ms between peaks (prevents double-counting)
- Adaptive threshold: on peak detection, set to `0.6 * peakAmplitude + 0.4 * currentThreshold`
- Threshold decay: multiply by 0.995 per sample when no peak detected (at 64 Hz this decays ~32%/sec)

**CRITICAL warmup:** For the first 2 seconds (128 samples), run the filter but do NOT detect peaks. After warmup, set initial threshold to 75th percentile of the absolute filtered signal amplitude seen during warmup. This prevents false peaks from threshold starting at zero.

**3. Artifact Rejection (two-tier, matching HRMAdapter)**

Same logic as HRMAdapter:
- Tier 1 (absolute bounds): reject IBI outside 300-2000ms
- Tier 2 (relative): maintain 5-beat median history, reject if IBI deviates >20% from median
- On rejection: do NOT write to rrBuffer. Increment artifact counter for signal quality.

**4. Signal Quality Indicator**

Track a rolling 30-second window of peak attempts vs artifact rejections:
- `good`: <10% rejection rate
- `fair`: 10-30% rejection rate
- `poor`: >30% rejection rate
Update `AppState.ppgSignalQuality` every 5 seconds.

**5. RR Buffer Write**

On valid (non-rejected) IBI:
```javascript
AppState.rrBuffer[AppState.rrHead % 512] = ibi;
AppState.rrHead++;
AppState.bpm = Math.round(60000 / ibi);
AppState.beats++;
```
This is identical to how HRMAdapter writes — the DSP engine is source-agnostic.

**6. Channel Selection**

Accept raw PPG samples from a single channel (configurable). Default to channel 1 (green) based on research suggesting green PPG has best cardiac signal. Expose `setPPGChannel(index)` for the calibration task to change if needed.
