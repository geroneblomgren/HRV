<task type="auto">
  <name>Task 1: Create js/audio.js — Lookahead scheduler and three tone synthesizers</name>
  <files>js/audio.js</files>
  <action>
Create js/audio.js as an ES module with the following exports and internal structure:

**Exports:**
- `initAudio()` — Creates AudioContext and master GainNode. MUST be called inside a user gesture handler (startSession click chain). If already created, calls `ctx.resume()` if suspended. Master gain default 0.4.
- `startPacer(pacingFreqHz)` — Computes halfPeriod = 1/(pacingFreqHz*2). Sets _nextCueTime = ctx.currentTime + 0.1. Sets _nextPhase = 'inhale'. Starts the scheduler tick loop.
- `stopPacer()` — Clears the setTimeout timer. Does NOT close AudioContext (reused on next session).
- `setStyle(style)` — Sets module-level _currentStyle to 'pitch'|'swell'|'bowl'. Takes effect on next scheduled cue.
- `setVolume(value)` — 0.0 to 1.0. Uses `_masterGain.gain.setTargetAtTime(value, ctx.currentTime, 0.05)` for click-free transition.
- `getAudioTime()` — Returns `_ctx ? _ctx.currentTime : 0`. Renderer needs this for visual sync.

**Scheduler (internal):**
- Constants: LOOKAHEAD_MS = 25, SCHEDULE_AHEAD_SEC = 0.1
- `_schedulerTick(halfPeriod)`: While `_nextCueTime < ctx.currentTime + SCHEDULE_AHEAD_SEC`, call `_scheduleCue(...)`, write `AppState.nextCueTime = _nextCueTime` and `AppState.nextCuePhase = _nextPhase`, advance _nextCueTime += halfPeriod, toggle _nextPhase. Schedule next tick via `setTimeout(() => _schedulerTick(halfPeriod), LOOKAHEAD_MS)`.

**_scheduleCue(time, phase, halfPeriod) dispatches to current style:**

**Style 1 — Pitch (_schedulePitchCue):**
- OscillatorNode type 'sine'
- Frequency: inhale ramps 220 -> 350 Hz, exhale ramps 350 -> 220 Hz
- Use `osc.frequency.setValueAtTime(start, time)` then `osc.frequency.exponentialRampToValueAtTime(end, time + halfPeriod)`
- Gain envelope: `setValueAtTime(0, time)`, `linearRampToValueAtTime(0.6, time + 0.2)` (200ms attack), `linearRampToValueAtTime(0, time + halfPeriod)` (fade to silence)
- IMPORTANT: linearRamp to 0 is valid; exponentialRamp to 0 is NOT
- Connect osc -> gain -> _masterGain
- `osc.start(time)`, `osc.stop(time + halfPeriod + 0.05)`

**Style 2 — Swell (_scheduleSwellCue):**
- OscillatorNode type 'sine', constant frequency 180 Hz
- Gain: inhale swells 0.01 -> 0.7 over first half of halfPeriod, then fades 0.7 -> 0.01 over second half. Exhale: 0.01 -> 0.4 -> 0.01 (quieter).
- Use `linearRampToValueAtTime` for all gain changes
- Connect osc -> gain -> _masterGain
- `osc.start(time)`, `osc.stop(time + halfPeriod + 0.05)`

**Style 3 — Bowl (_scheduleBowlCue):**
- Singing bowl character: warm tone with fast attack and long exponential decay
- OscillatorNode type 'sine'. Inhale: 220 Hz. Exhale: 174 Hz (different pitches for the two bowl strikes).
- Gain: `setValueAtTime(0, time)`, `linearRampToValueAtTime(0.8, time + 0.03)` (30ms strike attack), `setTargetAtTime(0.001, time + 0.03, 0.8)` (timeConstant=0.8s for ~3s resonant decay tail)
- Connect osc -> gain -> _masterGain
- `osc.stop(time + 5)` — let decay ring out fully. Overlapping bowl cues are acceptable (real bowls ring together).

**Critical constraints:**
- Import AppState from './state.js'
- Every OscillatorNode is one-shot: create fresh for each cue, never reuse
- All audio param scheduling uses AudioContext time (not Date.now)
- No module-level AudioContext creation — only inside initAudio()
