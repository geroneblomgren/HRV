  <action>
Create `js/oura.js` as an ES module following the CDN ESM pattern (no build tools, no external libraries).

**Constants:**
- `OURA_API_BASE = 'https://api.ouraring.com'`
- `OURA_AUTH_URL = 'https://cloud.ouraring.com/oauth/authorize'`
- `OURA_TOKEN_URL = 'https://api.ouraring.com/oauth/token'`
- `REDIRECT_URI = 'http://localhost:5000/callback'`
- `CACHE_TTL_MS = 6 * 60 * 60 * 1000` (6 hours)
- Module-level `_apiBase` variable that defaults to `OURA_API_BASE` but can be switched to `'http://localhost:5001'` if CORS proxy is needed. Export a `setProxyBase(url)` function to toggle this.

**Token storage (via storage.js):**
- `getStoredToken()`: calls `getSetting('ouraToken')` — returns `{access_token, refresh_token?, expires_at?}` or undefined
- `storeToken(tokenObj)`: calls `setSetting('ouraToken', tokenObj)`

**PAT-first auth:**
- `tryPatAuth(key)`: `GET /v2/usercollection/personal_info` with `Authorization: Bearer ${key}`. Returns `true` if 200, `false` otherwise. On success, call `storeToken({access_token: key})` and set `AppState.ouraConnected = true`.

**PKCE helpers (Web Crypto, no library):**
- `generateCodeVerifier()`: 32 random bytes -> base64url
- `generateCodeChallenge(verifier)`: SHA-256 via `crypto.subtle.digest` -> base64url
- `launchPkceFlow(clientId)`: Generate verifier+challenge, store verifier in `sessionStorage`, redirect to Oura auth URL with `response_type=code`, `scope='daily sleep'`, `code_challenge_method=S256`
- `handleCallback()`: Read `code` and `state` from `window.location.search`. Verify state matches. POST to token URL with `grant_type=authorization_code`, `code_verifier`, `redirect_uri`. Store returned token. Clear URL params with `history.replaceState`. Return token object.

**HRV data fetch:**
- `getHrvData(token?)`:
  1. If no token arg, call `getStoredToken()` to retrieve
  2. Check cache via `getOuraCache()` — if fresh (< 6h), return `cached.data`
  3. Fetch `GET /v2/usercollection/sleep?start_date={30d ago}&end_date={today}` with Bearer token
  4. Use `_apiBase` (not hardcoded OURA_API_BASE) so proxy works transparently
  5. Filter response for `type === 'long_sleep'` to avoid naps (fall back to longest `total_sleep_duration` if no `long_sleep` for a day)
  6. Map to `[{day: 'YYYY-MM-DD', hrv: number}]`
  7. Cache via `setOuraCache(hrvArray)` and set `AppState.ouraData = hrvArray`
  8. Return the array

**Token refresh (lazy check):**
- Before each `getHrvData` call, check if `expires_at` is past. If so and `refresh_token` exists, POST to token URL with `grant_type=refresh_token`. Update stored token. If refresh fails, set `AppState.ouraConnected = false` and return null (dashboard will show reconnect prompt).

**Exports:** `tryPatAuth`, `launchPkceFlow`, `handleCallback`, `getHrvData`, `getStoredToken`, `storeToken`, `setProxyBase`

**Key pitfall avoidances:**
- Filter for `long_sleep` type to avoid nap records with incorrect HRV (Research pitfall 4)
- Use `_apiBase` variable so CORS proxy is transparent to all fetch calls
- Store token in IndexedDB via `setSetting`, NOT localStorage (security per Research)
- Request both `daily sleep` scopes in OAuth2 (Research open question 3)
