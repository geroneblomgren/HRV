**js/dashboard.js — full module:**

Create as ES module. Imports: `getHrvData`, `getStoredToken`, `storeToken`, `tryPatAuth` from `./oura.js`; `querySessions` from `./storage.js`; `AppState` from `./state.js`.

Module state:
- `_initialized = false` (prevent double init)
- `_canvas`, `_ctx` (chart Canvas refs)
- `_hrvData = []`, `_sessionData = []` (current data)
- `_rangeDays = 30` (selected range)
- `_hitTargets = []` (array of {x, y, type, data} for tooltip hit detection)

**`initDashboard()` (exported):**
1. If `_initialized` and data is fresh, just return (idempotent)
2. Get stored token via `getStoredToken()`
3. If no token: show `#dashboard-connect`, hide `#dashboard-content`. Wire "Connect" button click to call `tryPatAuth(inputValue)`. On success, transition to dashboard view. On failure, show hint that it may be a client ID and offer PKCE flow.
4. If token exists: show `#dashboard-loading`, hide connect prompt. Call `getHrvData()`. On success, call `_renderDashboard()`. On error (e.g., expired token), show reconnect prompt.
5. Set `_initialized = true`

**`_renderDashboard()`:**
1. Hide loading, show `#dashboard-content`
2. Call `_computeMetrics()` and update the 4 metric card DOM elements
3. Call `_setupChart()` and `_drawChart()`

**`_computeMetrics(rangeDays)` — derive the 4 cards:**
1. **Tonight HRV**: Last entry in `_hrvData` array. Trend arrow: compare to 7-day average. Up if > 5% above, down if > 5% below, flat otherwise.
2. **Practice Streak**: Count consecutive days backward from today that have at least one session in `_sessionData`. Use date strings for comparison.
3. **Avg Coherence 7d**: Filter `_sessionData` to last 7 days, average `meanCoherence`. Handle days with multiple sessions by averaging all.
4. **HRV Trend**: Compare 7-day HRV average to 30-day HRV average. "Improving" if 7d > 30d by 5%+, "Declining" if 7d < 30d by 5%+, "Stable" otherwise.

**`_setupChart()`:**
- Get canvas element, size it to parent using same DPR pattern as `setupCanvas()` in renderer.js (copy the pattern, do not import — renderer.js does not export it)
- Store ctx and dimensions

**`_drawChart()`:**
- Clear canvas
- Define padding: `{top: 30, bottom: 50, left: 60, right: 60}`
- Compute X scale: `xToPixel(dayStr)` maps date to pixel position within chart area, based on `_rangeDays`
- Compute Y scales:
  - HRV (left axis): auto-range from actual data with 10% padding (do NOT hardcode 20-80ms per Research anti-pattern warning)
  - Coherence (right axis): fixed 0-100
- Draw grid lines: horizontal dashed lines for both axes, vertical lines for date ticks (every 7 days, or adaptive based on range)
- Draw date labels along bottom axis (format: "Mar 1", "Mar 8", etc.)
- Draw left axis labels (HRV in ms, teal color `#14b8a6`)
- Draw right axis labels (Coherence 0-100, purple color `#a78bfa`)
- Draw HRV teal line: `ctx.strokeStyle = '#14b8a6'`, lineWidth 2, connect all HRV points with lineTo. Use bezier curves for smoothing (quadraticCurveTo with midpoints).
- Draw coherence dots: `ctx.fillStyle = '#a78bfa'`, radius 5px circles at each session day. Store each drawn point in `_hitTargets` array for tooltip hit detection.
- Also store HRV line points in `_hitTargets`
- Draw axis titles: "HRV (ms)" rotated on left, "Coherence" rotated on right

**`_getSessionsByDay(rangeDays)`:**
- Call `querySessions({limit: rangeDays * 3})`
- Group by day (ISO date string from `session.date`)
- Average `meanCoherence` for days with multiple sessions
- Return `[{day, meanCoherence, durationSeconds}]`

**Tooltip system:**
- On canvas `mousemove`: call `_findNearest(mouseX, mouseY)` searching `_hitTargets` within 15px radius
- If found: position `#dashboard-tooltip` at mouse position (offset 10px right, 10px up), populate with date, value, type-specific info (HRV ms or coherence score + duration)
- If not found: hide tooltip
- On canvas `mouseleave`: hide tooltip

**Range button handling:**
- Wire click on each `.range-btn`: set `_rangeDays`, toggle `.active` class, re-fetch session data for new range, call `_drawChart()` and `_computeMetrics()`

**Window resize:**
- On `resize`, if dashboard is visible, re-setup canvas and redraw

**Key implementation details per user decisions:**
- Rest days show gap in coherence dots; HRV line continues (no interpolation needed, just connect available points)
- No session history list — chart and metrics only
- Tooltips show: date, HRV value (if HRV point), coherence score + session duration (if session point)
- Chart redraws only on: data update, range change, window resize (NOT on rAF)
