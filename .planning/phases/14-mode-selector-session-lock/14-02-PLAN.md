Build the Phase 14 UI layer on top of the session-lock state primitives from Plan 14-01.

Specifically:
1. Add a segmented-pill Mode Picker at the top of the Practice panel body (#tab-practice) with three pills: Standard (default) · Pre-Sleep · Meditation. Style by cloning the existing `.duration-picker` / `.duration-btn` rules (D-02).

4. Wire click handlers in main.js: clicking a pill writes `AppState.sessionMode`, persists via `saveSelectedMode()`, and swaps which mode-panel is visible (by toggling a `.active-mode-panel` class or equivalent CSS-driven visibility).
5. Subscribe to sessionPhase in main.js: when isSessionActive() becomes true, disable all 3 mode pills, all 4 start buttons (discovery-start-btn, practice-start-btn, pre-sleep-start-btn, meditation-start-btn), and all 3 nav tabs. When sessionPhase returns to 'idle', re-enable everything.
6. Restore selected mode on page load via `loadSelectedMode()` — Plan 14-01's state.js already does this for AppState.sessionMode, but main.js must use the value to correctly render which pill/panel is active at startup.

Fully unblocks Phase 16 (Pre-Sleep) and Phase 18 (Meditation), which will only need to replace their respective placeholder mode-panel's contents with real implementations.
