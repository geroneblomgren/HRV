  <action>
Create `js/paceController.js` — the closed-loop pace adjustment module.

**Exports:**
- `initPaceController(tunedFreqHz)` — called once at session start
- `paceControllerTick(sessionElapsedSec)` — called every 1 second from DSP tick

**Constants:**
- `TRIGGER_THRESHOLD = 50` — phase lock below this triggers adjustment
- `TRIGGER_SECONDS = 10` — consecutive seconds below threshold before activating
- `MAX_RATE_HZ = 0.01 / 30` — max Hz change per second (~0.00033 Hz/tick)
- `MAX_OFFSET_BPM = 0.5` — hard bound: ±0.5 BPM from tuned frequency
- `SMOOTHING_WINDOW = 5` — seconds of PSD peak readings for median smoothing
- `MIN_RELATIVE_POWER = 0.05` — skip PSD peak if < 5% of total LF band power (Pitfall 4 from research)

**Module state:**
- `_tunedFreqHz` — set at init, never changes
- `_belowThresholdSec` — counter for consecutive seconds below threshold
- `_prevPhaseLockScore` — for uptrend detection
- `_userFreqHistory` — circular buffer for PSD peak smoothing (max SMOOTHING_WINDOW entries)
- `_active` — whether controller is currently adjusting

**initPaceController(tunedFreqHz):**
- Store `tunedFreqHz` as `_tunedFreqHz`
- Reset all module state to defaults
- Set `AppState.paceControllerActive = false`
- Set `AppState.pacerAtBound = false`
- Set `AppState.pacingFreqTuned = tunedFreqHz`

**paceControllerTick(sessionElapsedSec):**
1. Guard: return immediately if `sessionElapsedSec < 120` (calibration gate — spectralBuffer is null before this)
2. Read `AppState.phaseLockScore` as `currentScore`
3. Detect uptrend: `isRising = currentScore > _prevPhaseLockScore`; update `_prevPhaseLockScore = currentScore`
4. If `currentScore >= TRIGGER_THRESHOLD`: reset `_belowThresholdSec` to 0, set `_active = false`, set `AppState.paceControllerActive = false`, return (lock recovered — keep current frequency)
5. If `isRising`: return without incrementing counter (pause on uptrend — user is catching up)
6. Increment `_belowThresholdSec`
7. If `_belowThresholdSec < TRIGGER_SECONDS`: return (not triggered yet)
8. Read `AppState.spectralBuffer` — if null, return
9. Detect user's breathing rate from PSD peak in LF band (0.04-0.15 Hz):
   - Find peak bin using same logic as `findPeakBin` in dsp.js (inline the 10-line function — don't widen dsp.js public API per research recommendation)
   - Convert peak bin to Hz
   - **Power guard (Pitfall 4):** compute power around detected peak (±0.01 Hz) vs total LF band power. If relative power < MIN_RELATIVE_POWER, skip this tick (don't poison the median buffer with noise)
10. Push detected Hz to `_userFreqHistory`, shift if > SMOOTHING_WINDOW
11. Compute median of `_userFreqHistory` as `smoothedUserHz`
12. Compute hard bounds: `maxHz = _tunedFreqHz + MAX_OFFSET_BPM / 60`, `minHz = _tunedFreqHz - MAX_OFFSET_BPM / 60`
13. Clamp target: if `smoothedUserHz > maxHz` → target = maxHz (clamped), if < minHz → target = minHz (clamped)
14. Step toward target by MAX_RATE_HZ: `delta = targetHz - currentHz`, `step = sign(delta) * min(abs(delta), MAX_RATE_HZ)`, `newHz = currentHz + step`
15. Write `AppState.pacingFreq = newHz`
16. Set `AppState.paceControllerActive = true`
17. Set `AppState.pacerAtBound = clamped && abs(smoothedUserHz - newHz) > MAX_RATE_HZ` (amber badge when at bound AND user is further away)

**Helper functions (private):**
- `_findPSDPeak(psd, sampleRate, fftSize)` — finds peak bin in 0.04-0.15 Hz range. Inline the findPeakBin logic from dsp.js.
- `_median(arr)` — sort-and-pick-middle on small array
- `_hzToBin(hz, sampleRate, fftSize)` and `_binToHz(bin, sampleRate, fftSize)` — Hz/bin conversion. Either export from dsp.js or inline (both approaches are fine; check if dsp.js already exports these, and if so, import them instead of duplicating).

**Note on dsp.js:** Check if `hzToBin` and `binToHz` are currently exported. If yes, import them. If not, either export them from dsp.js (preferred — small public API addition) or inline the conversion formulas (`bin = Math.round(hz * fftSize / sampleRate)` and `hz = bin * sampleRate / fftSize`). Also check the FFT size and sample rate used (likely 4Hz sample rate and 256-point FFT from the DSP config).
